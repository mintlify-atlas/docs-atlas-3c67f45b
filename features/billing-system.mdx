The billing system automatically calculates water consumption charges based on meter readings and applies appropriate rates based on property type.

## Billing Cycle Overview

Billing cycles are created monthly for all active clients with verified meter readings.

<Steps>
  <Step title="Meter Reading Collection">
    Meter readers collect current readings from all active clients during the billing period.
    
    Each reading includes:
    - Current meter reading
    - Previous meter reading
    - Reading date
    - Meter reader ID
  </Step>

  <Step title="Consumption Calculation">
    The system automatically calculates consumption:
    
    ```
    Consumption (m³) = Current Reading - Previous Reading
    ```
    
    <Info>
    Consumption is measured in cubic meters (m³) of water usage.
    </Info>
  </Step>

  <Step title="Rate Application">
    Appropriate rates are applied based on property type:
    - **Residential**: Lower rate for household consumption
    - **Commercial**: Higher rate for business properties
    
    The rate is retrieved from the `rates` table based on:
    - Property type (rate_type)
    - Current billing month
  </Step>

  <Step title="Bill Generation">
    Final bill amount is calculated:
    
    ```
    Billing Amount = Consumption × Rate
    Total Due = Billing Amount + Tax + Penalty (if applicable)
    ```
  </Step>
</Steps>

## Billing Data Structure

Each billing record contains comprehensive information:

<CodeGroup>
```sql billing_data table
CREATE TABLE `billing_data` (
  `id` int(11) NOT NULL AUTO_INCREMENT,
  `billing_id` varchar(50) NOT NULL,
  `client_id` varchar(20) NOT NULL,
  `meter_number` varchar(50) NOT NULL,
  `prev_reading` decimal(10,2) NOT NULL DEFAULT 0.00,
  `curr_reading` decimal(10,2) NOT NULL DEFAULT 0.00,
  `reading_type` varchar(20) DEFAULT NULL,
  `consumption` decimal(10,2) NOT NULL DEFAULT 0.00,
  `rates` decimal(7,2) NOT NULL DEFAULT 0.00,
  `rate_type` enum('Commercial','Residential') NOT NULL,
  `rate_fee_id` varchar(50) NOT NULL,
  `billing_amount` decimal(10,2) NOT NULL DEFAULT 0.00,
  `billing_status` enum('initial','unpaid','paid') DEFAULT 'initial',
  `billing_type` enum('initial','unverified','verified','billed','paid','disputed','canceled','overdue') NOT NULL DEFAULT 'initial',
  `penalty` decimal(7,2) NOT NULL DEFAULT 0.00,
  `billing_month` varchar(20) NOT NULL,
  `due_date` date DEFAULT NULL,
  `disconnection_date` date DEFAULT NULL,
  `period_to` date DEFAULT NULL,
  `period_from` date DEFAULT NULL,
  PRIMARY KEY (`id`)
);
```
</CodeGroup>

## Rate Types and Calculation

### Rate Management

Rates are configured in the system and can vary by:
- Property type (Commercial vs Residential)
- Billing month/period
- Tax percentage (fixed at 2%)

<Tabs>
  <Tab title="Residential Rates">
    Residential properties receive preferential rates for household water consumption.
    
    **Typical Structure:**
    - Base rate per cubic meter
    - Progressive pricing may apply for high usage
    - Fixed 2% tax
  </Tab>
  
  <Tab title="Commercial Rates">
    Commercial properties are charged higher rates due to business usage.
    
    **Typical Structure:**
    - Higher base rate per cubic meter
    - Flat rate regardless of consumption volume
    - Fixed 2% tax
  </Tab>
</Tabs>

<CodeGroup>
```sql rates table
CREATE TABLE `rates` (
  `id` int(11) NOT NULL AUTO_INCREMENT,
  `rate_fee_id` varchar(50) NOT NULL,
  `rate_type` varchar(50) NOT NULL,
  `rates` varchar(20) NOT NULL,
  `tax` smallint(6) NOT NULL DEFAULT 2,
  `billing_month` varchar(20) NOT NULL,
  `reference_id` varchar(20) NOT NULL,
  `time` time NOT NULL,
  `date` date NOT NULL,
  `timestamp` timestamp NOT NULL DEFAULT current_timestamp(),
  PRIMARY KEY (`id`)
);
```
</CodeGroup>

### Updating Rates

Administrators can update billing rates through the admin interface:

<CodeGroup>
```php admin/database_actions.php
case 'updateRates':
  handleUpdateRates($dbQueries);
  break;

function handleUpdateRates($dbQueries) {
  if (isset($_POST['formData'])) {
    $formData = $_POST['formData'];
    $updateResponse = $dbQueries->updateRates($formData);
    echo json_encode($updateResponse);
  }
}
```
</CodeGroup>

<Warning>
Rate changes typically apply to future billing cycles only. Existing unpaid bills retain their original rates unless manually adjusted.
</Warning>

## Penalties and Due Dates

### Due Date Calculation

Each bill has:
- **Billing Date**: When the bill is generated
- **Due Date**: Payment deadline (typically 15-30 days after billing)
- **Disconnection Date**: Service suspension date if unpaid (typically 15 days after due date)

<CodeGroup>
```sql billing_data fields
`due_date` date DEFAULT NULL,
`disconnection_date` date DEFAULT NULL,
`period_to` date DEFAULT NULL,
`period_from` date DEFAULT NULL,
```
</CodeGroup>

### Penalty Application

Penalties are automatically applied to overdue bills:

<Steps>
  <Step title="Check Due Date">
    System checks if current date exceeds the due date
  </Step>
  
  <Step title="Calculate Penalty">
    Penalty is calculated based on:
    - Days overdue
    - Penalty rate configured in `penalty_fees` table
    - Original billing amount
  </Step>
  
  <Step title="Update Bill">
    The penalty amount is added to the total due:
    ```
    Total Amount Due = Billing Amount + Tax + Penalty
    ```
  </Step>
</Steps>

<CodeGroup>
```php admin/database_actions.php
case 'updatePenaltyFees':
  handleUpdatePenaltyFees($dbQueries);
  break;
```
</CodeGroup>
