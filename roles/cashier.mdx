## Overview

The Cashier portal (`/cashier/`) is the financial gateway of the Roxas Water Billing System. Cashiers are responsible for confirming payments from clients, generating receipts, and notifying administrators of completed transactions. This role ensures accurate payment processing and maintains financial records.

**Location**: `application_payments.php`

Process one-time application fee payments for new client connections:

#### Payment Workflow

<Steps>
  <Step title="Review Confirmed Applications">

  </Step>
  <Step title="Verify Payment Amount">

  <Step title="Enter Payment Details">
    Input amount paid and any payment reference information

```php database_queries.php:613
{

    $amountPaid = htmlspecialchars($formData['amountPaid'], ENT_QUOTES, 'UTF-8');

    $transactionType = 'application_payment';
    
    // Retrieve application fee breakdown
    $clientApplicationFees = $this->retrieveClientApplicationFees();
    $amountDue = intval($applicationFee) + intval($inspectionFee) + 
                 intval($registrationFee) + intval($connectionFee) + 
                 intval($installationFee);
    
    // Calculate change
    $remainingBalance = intval($amountPaid) - intval($amountDue);
    
    // Insert transaction record
    $this->insertIntoTransactions($transactionData);
    
    // Update application billing status
    $this->updateClientApplication($applicationID, $applicationFeeID);
    
    // Notify administrator
    $this->addNotification($userID, $message, 'payment_confirmation', $applicationID);
}
```

**Fee Components** (configured by Admin):
- Application Fee
- Inspection Fee  
- Registration Fee
- Connection Fee
- Installation Fee

<Note>
The cashier views the current fee structure from `client_application_fees` table. These fees are set by administrators and apply to all pending applications.
</Note>

```php database_queries.php:432
public function confirmBillingPayment($formData)
{
    $clientID = htmlspecialchars($formData['clientID'], ENT_QUOTES, 'UTF-8');
    $amountPaid = htmlspecialchars($formData['amountPaid'], ENT_QUOTES, 'UTF-8');

    $billingAmount = $billingData['billingData']['billing_amount'];
    $penalty = $billingData['billingData']['penalty'];
    
    // Get current tax rate
    $rates = $this->retrieveBillingRates();
    $taxRate = $rates['rates']['tax'];
    
    // Calculate total
    $calculatedTotalBill = $this->calculateTotalBill($billingAmount, $penalty, $taxRate);
    $taxAmount = $calculatedTotalBill[0];
    $totalWithTax = $calculatedTotalBill[1];
    
    // Insert transaction
    $this->insertIntoTransactions($transactionData);
    
    // Update billing status
    $this->updateBillingData($billingID);
    
    // Generate and email receipt
    $pdfGenerator->generateBillingReceipt($receiptData);
    $wbsMailer->handleEmailSend($clientData, $emailPath);
}
```
</CodeGroup>

**Bill Calculation**:
```
Base Amount = Consumption (m³) × Rate (per m³)
Penalty = Late payment fees (if applicable)
Tax = Base Amount × 12%
Total Due = Base Amount + Penalty + Tax
Change = Amount Paid - Total Due
```

```php database_queries.php:60
public function generateBillingReceipt($receiptData)
{
    // Initialize Dompdf with options
    $dompdf = new Dompdf($options);
    $dompdf->setPaper(array(0, 0, 504, 504), 'portrait');
    
    // Generate QR code for transaction
    $qrCode = new QrCode($receiptData['no']);
    $writer = new PngWriter();
    $result = $writer->write($qrCode);
    $qrDataUri = $result->getDataUri();
    
    // Load receipt template
    $template = file_get_contents('templates/invoice-billing.html');
    $invoiceHtml = str_replace(
        ['{{tx_qr}}', '{{transaction_id}}', '{{account_no}}', ...],
        [$qrDataUri, $no, $accountNo, ...],
        $template
    );
    
    // Generate PDF
    $dompdf->loadHtml($invoiceHtml);
    $dompdf->render();
    
    // Save to temp directory
    $filename = $no . '.pdf';
    $filepath = 'temp/receipts/' . $filename;
    file_put_contents($filepath, $dompdf->output());
}
```
</CodeGroup>

<Tip>
Receipts are automatically emailed to the client's registered email address using the `WBSMailer` class, which integrates with Gmail SMTP.
</Tip>

### 4. Transaction Recording

**Location**: Handled via `insertIntoTransactions()` in `database_queries.php`

Every payment confirmation creates a permanent transaction record:

**Transaction Data**:
- `transaction_id`: Unique timestamp-based ID
- `transaction_type`: "application_payment" or "bill_payment"
- `reference_id`: Application ID or Billing ID
- `client_id`: Client account (blank for applications)
- `transaction_desc`: Human-readable description
- `amount_due`: Total amount owed
- `amount_paid`: Amount received from client
- `remaining_balance`: Change returned to client
- `confirmed_by`: Cashier user ID
- `time`, `date`, `timestamp`: Payment datetime

<Info>
Transaction IDs are generated using the format: `[Unix Timestamp][4-digit Random Number]`, ensuring uniqueness across all payments.
</Info>

## Best Practices

<Check>
**Verify Amount**: Always confirm the amount paid matches the total due before confirming payment.
</Check>

<Tip>
**Receipt Delivery**: Verify client email addresses before confirming payment to ensure receipt delivery.
</Tip>

<Warning>
**No Duplicates**: The system prevents duplicate payments by checking if a transaction already exists for the reference ID. If a client claims non-receipt, check the transaction log before attempting to reprocess.
</Warning>

## Error Handling

**Common Issues**:

1. **Duplicate Payment**: "Failed to insert new transaction. [Client] already paid, please check transaction history."
   - **Solution**: Verify in transaction history. If payment was successful, regenerate receipt.

2. **Email Failure**: "Failed to send receipt to email."
   - **Solution**: Transaction still recorded. Manually email receipt from temp/receipts/ directory.

3. **Invalid Client ID**: "No data found."
   - **Solution**: Verify client ID format and billing status. Client may not have unpaid bills.

## Technical Details

**Payment Processing Flow**:
1. User clicks "Payment" button with client/application ID
2. System retrieves billing/application data
3. Modal displays amount breakdown
4. Cashier enters amount paid
5. System validates and calculates change
6. Transaction inserted (with rollback on failure)
7. PDF receipt generated
8. Email sent to client
9. Notification sent to admin (if application payment)
10. Success message displayed

**Database Classes**:
- `DatabaseQueries`: Main query handler
- `PdfGenerator`: Receipt PDF creation
- `WBSMailer`: Email delivery via PHPMailer

**Session Data**:
- `user_id`: Cashier identifier (recorded in transactions)
- `user_name`: Display name (shown on receipts)
- `user_role`: "cashier" role designation
