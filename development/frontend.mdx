### Build process

Tailwind CSS is compiled using the npm script:

```bash package.json
npm run dev
```

This watches for changes and compiles:
```
src/input.css → assets/css/style.css
```

<Info>
The compiled CSS file is included in all portal layouts via `layouts/links.php`.
</Info>

## JavaScript modules

The system uses modular JavaScript for different features:

### DataTableWithPagination.js

Handles table pagination, search, and filtering:

```javascript
class DataTableWithPagination {
  constructor(config) {
    this.tableBody = document.querySelector(config.tableBodySelector);
    this.paginationContainer = document.querySelector(config.paginationSelector);
    this.searchInput = document.querySelector(config.searchSelector);
    this.itemsPerPage = config.itemsPerPage || 10;
    this.currentPage = 1;
    this.filteredData = [];
    this.allData = [];
  }

  init(data) {
    this.allData = data;
    this.filteredData = data;
    this.render();
    this.attachEventListeners();
  }
}
```

### AddressFetcher.js

Dynamically loads barangay addresses:

```javascript
class AddressFetcher {
  constructor(selectElement) {
    this.selectElement = selectElement;
  }

  async fetchAddresses() {
    const response = await fetch('database_actions.php?action=fetch_addresses');
    const data = await response.json();
    this.populateSelect(data);
  }
}
```

### clientActions.js

Handles client CRUD operations:

```javascript
function addClient(formData) {
  $.ajax({
    url: 'database_actions.php',
    type: 'POST',
    data: {
      action: 'add_client',
      ...formData
    },
    success: function(response) {
      // Handle success
    },
    error: function(error) {
      // Handle error
    }
  });
}
```

### chartsData.js

Fetches and prepares data for ApexCharts:

```javascript
function fetchChartData() {
  fetch('charts_data.php')
    .then(response => response.json())
    .then(data => {
      renderPieChart(data.clientDistribution);
      renderBarChart(data.monthlyRevenue);
      renderLineChart(data.consumption);
    });
}
```

## Form validation

The system uses validate.js for client-side validation:

```javascript application_validate.js
const constraints = {
  first_name: {
    presence: { message: "First name is required" },
    length: {
      minimum: 2,
      maximum: 50
    }
  },
  email: {
    presence: true,
    email: true
  },
  phone_number: {
    presence: true,
    format: {
      pattern: /^[0-9]{11}$/,
      message: "must be 11 digits"
    }
  }
};

const errors = validate(formData, constraints);
if (errors) {
  displayErrors(errors);
}
```

## Best practices

<Accordion title="Use Tailwind utilities over custom CSS">
Prefer Tailwind classes for consistency and maintainability. Only write custom CSS when absolutely necessary.
</Accordion>

<Accordion title="Modularize JavaScript">
Keep JavaScript organized in separate files by feature. Use classes for reusable components.
</Accordion>

<Accordion title="Validate client-side and server-side">
Always validate on both the client (for UX) and server (for security).
</Accordion>

<Accordion title="Use semantic HTML">
Proper HTML structure improves accessibility and SEO.
</Accordion>
