## Server Requirements

The Roxas Water Billing System requires the following server environment:

### PHP Requirements

- **PHP Version**: 7.4 or higher (8.0+ recommended)
- **Required Extensions**:
  - `mbstring` - Multibyte string processing (required for dompdf)
  - `gd` - Image processing (required for dompdf PDF generation)
  - `mysqli` - MySQL database connectivity
  - `json` - JSON data handling
  - `curl` - HTTP requests (for OAuth and email)
  - `openssl` - Secure connections
  - `fileinfo` - File upload handling

### Database Requirements

- **MySQL**: 5.7 or higher
- **MariaDB**: 10.4+ (tested with 10.4.28)

### Web Server

- Apache 2.4+ or Nginx
- Document root should point to the application root directory
- `.htaccess` support (for Apache) or equivalent Nginx configuration

## PHP Configuration

<Warning>
  The `mbstring` and `gd` extensions are **critical** for PDF generation. Without these, the dompdf library will fail to generate invoices and reports.
</Warning>

Edit your `php.ini` file to enable required extensions:

```ini php.ini
; Enable mbstring extension for multibyte string processing
extension=mbstring

; Enable GD extension for image processing and PDF generation
extension=gd

; Other recommended settings
memory_limit = 256M
upload_max_filesize = 10M
post_max_size = 10M
max_execution_time = 120
```

Restart your web server after modifying `php.ini`:

```bash
# Apache
sudo systemctl restart apache2

# Nginx with PHP-FPM
sudo systemctl restart php-fpm
sudo systemctl restart nginx
```

## Composer Dependencies

<Steps>
  <Step title="Install Composer">
    Ensure Composer is installed on your system:

    ```bash
    composer --version
    ```

    If not installed, download from [getcomposer.org](https://getcomposer.org/)
  </Step>

  <Step title="Install Admin Dependencies">
    Navigate to the admin directory and install dependencies:

    ```bash
    cd admin
    composer install
    ```

    This installs:
    - `dompdf/dompdf` ^2.0 - PDF generation for invoices and reports
    - `phpmailer/phpmailer` ^6.8 - Email functionality
    - `league/oauth2-google` ^4.0 - Google OAuth authentication
  </Step>

  <Step title="Install Cashier Dependencies">
    ```bash
    cd cashier
    composer install
    ```

    Installs:
    - `dompdf/dompdf` ^2.0
    - `phpmailer/phpmailer` ^6.8
    - `endroid/qr-code` ^5.0 - QR code generation for receipts
    - `league/oauth2-google` ^4.0
  </Step>

  <Step title="Install Meter Reader Dependencies">
    ```bash
    cd meter-reader
    composer install
    ```

    Installs the same dependencies as the cashier module.
  </Step>
</Steps>

## NPM Dependencies

The system uses Tailwind CSS and several JavaScript libraries for the frontend.

<Steps>
  <Step title="Install Node.js">
    Ensure Node.js 14+ and npm are installed:

    ```bash
    node --version
    npm --version
    ```
  </Step>

  <Step title="Install Dependencies">
    From the project root directory:

    ```bash
    npm install
    ```

    This installs:

    **Production Dependencies:**
    - `apexcharts` ^3.45.1 - Charts and data visualization
    - `flowbite` ^1.8.1 - Tailwind CSS components
    - `flowbite-datepicker` ^1.2.2 - Date picker component
    - `jquery` ^3.7.1 - DOM manipulation and AJAX
    - `validate.js` ^0.13.1 - Client-side form validation

    **Development Dependencies:**
    - `tailwindcss` ^3.3.5 - CSS framework
    - `@tailwindcss/forms` ^0.5.3 - Form styling plugin
  </Step>
</Steps>

The project uses Tailwind CSS with custom configuration and Flowbite components.

### Building CSS

Run Tailwind in watch mode during development:

```bash
npm run dev
```

This watches `./src/input.css` and outputs to `./assets/css/style.css`.

For production builds, use:

```bash
tailwindcss -i ./src/input.css -o ./assets/css/style.css --minify
```

## Common Issues

### Missing mbstring Extension

If you see errors about mbstring:

```bash
# Ubuntu/Debian
sudo apt-get install php-mbstring

# CentOS/RHEL
sudo yum install php-mbstring

# Restart Apache
sudo systemctl restart apache2
```

### Missing GD Extension

```bash
# Ubuntu/Debian
sudo apt-get install php-gd

# CentOS/RHEL
sudo yum install php-gd
```

### Composer Memory Limit

If Composer runs out of memory:

```bash
php -d memory_limit=-1 $(which composer) install
```
